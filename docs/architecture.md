# Архитектура ЭкоВыхухоль Server

## Принципы

- модульный монолит на Django;
- API версионируется через `api/v1`;
- домены разделены по папкам `apps/*`;
- клиентская логика держится тонкой: бизнес-данные и авторизация живут на сервере.

## Домены

### `users`

- хранит профиль пользователя;
- принимает логин по email, телефону или username;
- отдает JWT и `me` endpoint для мобильного клиента.

### `posts`

- отдает публичную ленту и деталку поста;
- поддерживает пагинацию;
- хранит отдельную сущность `PostImage`, чтобы потом без перелома перейти от одного изображения к нескольким.

### `common`

- базовые mixin-модели;
- healthcheck и прочая общая инфраструктура.

## Почему такой сервер удобен для следующего этапа

- можно без боли добавить комментарии как отдельный app;
- можно подключить карту и мероприятия отдельными модулями;
- можно ввести роли, модерацию и админские сценарии без переписывания auth и feed.
